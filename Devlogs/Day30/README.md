\# Project V 30일차 개발일지



\---



\## 1. 작업 목표



기획서의 마물 포획 규칙과 현재 전투 보상 구조의 차이를 줄이기 위해 중복 마물 처리 방식을 수정한다. 중복 마물을 추가 경험치와 보유 수량으로 처리하지 않고 마물 강화 재료로 변환하도록 구현한다.



\---



\## 2. 구현 내용



\- 마물 강화 재료 진행 데이터 추가

\- 중복 포획 강화 재료 수량 설정

\- 중복 마물의 보유 수량 증가 제거

\- 중복 포획 경험치 보너스 제거

\- 중복 마물의 강화 재료 변환 구현

\- 전투 결과 데이터에 획득 강화 재료 저장

\- 보상 수령 후 결과 UI 갱신

\- 중복 변환 결과 문구 추가

\- 중복 변환 전투 로그 추가



\---



\## 3. 중복 포획 규칙



\- 처음 포획한 마물은 보유 마물 목록에 등록

\- 이미 보유한 마물은 강화 재료로 변환

\- 중복 마물은 보유 수량을 증가시키지 않음

\- 중복 포획 추가 경험치는 지급하지 않음

\- 중복 강화 재료는 임시로 5개 지급

\- 일반 전투 경험치는 기존처럼 모든 보유 마물에게 지급



\---



\## 4. 덱 규칙 확인



기존 덱 검증 구조를 변경하지 않고 다음 규칙이 유지되는지 확인했다.



\- 전투 덱 30장

\- 동일 카드 최대 3장

\- 빈 카드 데이터 차단

\- 빈 카드 ID 차단

\- 비정상 덱 전투 시작 차단



\---



\## 5. 테스트 내용



\- 신규 마물 포획

\- 중복 마물 포획

\- 중복 마물 강화 재료 변환

\- 강화 재료 보상 UI

\- 중복 변환 전투 로그

\- 보상 중복 수령 차단

\- 정상 30장 덱

\- 29장 덱 검증 실패

\- 동일 카드 4장 검증 실패

\- Unity Console 오류 확인



\---



\## 6. 작업 결과



중복 마물 포획 시 추가 경험치와 보유 수량을 지급하던 기존 구조를 강화 재료 변환 방식으로 수정했다. 전투 보상을 수령하면 획득한 강화 재료가 결과 UI와 전투 로그에 표시된다.



다중 마물 포획과 포획전 전용 규칙은 일반전과 포획전 데이터가 구현된 이후 추가한다.



\---



\## 7. 다음 개발 방향



31일차에는 OwnedCardData와 시작 보유 카드 목록을 구현한다. CardData는 카드의 고정 정보만 관리하고 보유 수량은 PlayerProgressManager에서 별도로 관리한다.
