\# Project V 29일차 개발일지

\## 1. 작업 목표



향후 게임 흐름 구현에 필요한 씬을 미리 준비하고, 하나의 파일에 집중되어 있던 BattleManager 코드를 기능별 partial 파일로 분리한다. 기존 전투 동작과 Inspector 연결을 유지하면서 코드 관리성과 가독성을 개선하는 것을 목표로 한다.



\---



\## 2. 씬 뼈대 구축



다음 씬을 새로 생성했다.



\- 00\_Bootstrap

\- 01\_MainMenu

\- 02\_DeckBuilder

\- 03\_StageSelect

\- 04\_Story



각 씬에는 향후 기능 구현에 사용할 기본 UI 오브젝트와 패널을 배치했다. 기존 전투 씬은 이름과 위치를 변경하지 않고 Build Profile에 함께 등록했다.



\---



\## 3. BattleManager 구조 분리



기존 BattleManager를 partial class로 변경하고 다음 파일로 기능을 분리했다.



\- BattleManagerDeck.cs

\- BattleManagerMonster.cs

\- BattleManagerHeroine.cs

\- BattleManagerUI.cs

\- BattleManagerResult.cs



BattleManager.cs에는 Inspector 연결 필드와 전투 초기화, 턴 전환 등 핵심 전투 흐름을 유지했다.

\## 6. 작업 결과



기존 전투 기능을 유지하면서 BattleManager의 기능을 여러 파일로 분리했다. 앞으로 각 기능을 수정할 때 하나의 긴 파일 전체를 탐색하지 않고 해당 기능 파일에서 작업할 수 있게 되었다.



또한 메인 메뉴, 덱 편성, 스테이지 선택, 스토리 씬의 기본 구조를 미리 준비해 이후 기능을 안전하게 연결할 수 있는 기반을 마련했다.
