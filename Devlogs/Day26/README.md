\# Project V 개발 일지 - Day 26

\## 개발 목표



전투 덱의 장수와 동일 카드 제한을 검사하고 드로우, 손패, 버린 카드, 재셔플 흐름을 정식 덱 규칙으로 통합한다.

\- 전투 덱 정확히 30장 규칙 구현

\- 동일 CardId 최대 3장 제한

\- 비어 있는 카드 데이터 검사

\- 비어 있는 CardId 검사

\- 잘못된 덱의 전투 시작 차단

\- 전투 시작 덱 셔플

\- Fisher-Yates 셔플 구현

\- 드로우 덱 소진 시 버린 카드 재셔플

\- 손패 최대 장수 제한

\- 드로우 불가 상황 로그 구현

\- Draw, Hand, Discard 수량 UI 구현

\- 현재 덱 장수 UI 구현

\- 카드 사용 후 덱 수량 갱신

\- 개발용 덱 검사 활성화 설정 추가



\## 덱 규칙



\- 전투 덱은 정확히 30장으로 구성한다.

\- 동일 CardId는 최대 3장까지 사용할 수 있다.

\- 동일 카드 복사본은 같은 CardData를 여러 번 덱에 넣는다.

\- 다른 종류의 카드는 서로 다른 CardId를 사용한다.

\- 손패는 최대 10장이다.

\- 드로우 덱이 비면 버린 카드를 섞어 새로운 드로우 덱으로 사용한다.

\- 손패에 남아 있는 카드는 재셔플 대상이 아니다.
