\# Project V 개발 일지



\## 4일차: 히로인 마물 공격 및 사망 처리 구현



\### 개발 목표



히로인이 필드에 소환된 마물을 우선 공격하고, 피해를 받은 마물의 HP 감소와 사망 및 필드 제거 기능을 구현한다.



\### 구현 내용



\- MonsterUnit에 마물 이름 반환 기능 추가

\- MonsterUnit에 사망 여부 확인 기능 추가

\- 마물 피해 처리 메서드 구현

\- 마물 HP가 0보다 작아지지 않도록 제한

\- 마물 피해 발생 시 HP UI 갱신

\- 히로인 공격 대상 결정 기능 구현

\- 필드에 마물이 존재하면 마물 우선 공격

\- 가장 먼저 소환된 마물을 첫 번째 공격 대상으로 설정

\- 필드에 마물이 없으면 플레이어 직접 공격

\- 마물 HP 0 도달 시 사망 처리

\- 사망한 마물을 필드 목록에서 제거

\- 사망한 마물 UI 오브젝트 제거

\- 마물 사망 후 남은 마물 자동 정렬

\- 마물 사망으로 확보된 필드에 새로운 마물 소환 허용

\- 플레이어 HP 0 도달 시 패배 처리

\- 전투 종료 후 카드와 턴 종료 버튼 비활성화

\### 테스트 결과



\- 히로인이 가장 먼저 소환된 마물을 공격하는 것을 확인했다.

\- 히로인 공격력만큼 마물 HP가 감소하는 것을 확인했다.

\- 변경된 마물 HP가 UI에 즉시 표시되는 것을 확인했다.

\- 마물 HP가 0보다 작아지지 않는 것을 확인했다.

\- 마물 HP가 0이 되면 필드에서 제거되는 것을 확인했다.

\- 사망한 마물의 UI 오브젝트가 제거되는 것을 확인했다.

\- 마물이 제거되면 남은 마물이 자동으로 정렬되는 것을 확인했다.

\- 가장 먼저 소환된 마물이 사망하면 다음 마물이 공격 대상이 되는 것을 확인했다.

\- 필드에 마물이 없을 때 플레이어가 직접 피해를 받는 것을 확인했다.

\- 플레이어 HP가 0이 되면 Defeat가 표시되는 것을 확인했다.

\- 전투 종료 후 카드와 턴 종료 버튼이 비활성화되는 것을 확인했다.

\- Unity Console Error 0건을 확인했다.



\### 현재 전투 흐름



1\. 플레이어가 마나를 사용하여 마물 카드를 사용한다.

2\. 카드에 연결된 마물이 필드에 소환된다.

3\. 플레이어가 턴 종료 버튼을 누른다.

4\. 필드의 모든 마물이 히로인을 자동으로 공격한다.

5\. 히로인은 가장 먼저 소환된 마물을 공격한다.

6\. 공격받은 마물의 HP가 감소한다.

7\. 마물 HP가 0이 되면 필드에서 제거된다.

8\. 필드에 마물이 없으면 히로인이 플레이어를 직접 공격한다.

9\. 다음 플레이어 턴에 마나가 회복되고 카드가 추가된다.
